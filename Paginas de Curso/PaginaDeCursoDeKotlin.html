Kotlin

Kotlin é uma linguagem de programação orientada a objetos, multiplataforma que foi desenvolvida pela JetBrains em 2011. Ela é uma linguagem totalmente orientada a objetos, mas com algumas características de linguagem funcional (como as funções lambda), com tipagem estática e executada pela Java Virtual Machine (Máquina Virtual do Java).

A Concepção do Projeto Kotlin

Kotlin foi criado em 2011 pela JetBrains, empresa famosa por suas IDE's como, IntelliJ IDEA, PHP Storm e Web Storm. O projeto foi iníciado pelo própria JetBrains e até hoje é permanece como um produto dela. O Kotlin foi criado para suprir uma necessidade que surgiu dentro da empresa em relação a outras linguagens pois não tinham tudo que eles precisavam. O projeto iria inicialmente ser totalmente feito com base em Scala, porém, o tempo de compilação da linguagem acabou provando ser muito mais lento em relação ao Java, por isso a empresa resolveu reunira os melhores aspectos de ambas as linguagens para compor o que hoje temos no Kotlin. Em 2012, esse projeto foi liberado para a comunidade no github, onde ele permanece até hoje como um projeto open source.

Por que utilizar Kotlin?

A ferramenta pode representar um ganho significativo em produtividade por ser uma linguagem intuitiva e simples de ser utilizada. Estima-se que o Kotlin pode levar a uma redução de cerca de até 40% do código que é utilizado para realizar representações em outras linguagens como Java, por exemplo. Java se trata de uma linguagem que demanda um extenso vocabulário para realizar esse processo e, apesar de existirem projetos para diminuir essa desvantagem, como o Lombok, por exemplo, ainda assim, ela não chega a ser tão eficaz quanto a expressividade que Kotlin consegue alcançar nativamente. A curva de aprendizagem do Kotlin é rápida e relativamente pequena, entretanto, escrever menos códigos significa que mais regras precisam ser lembradas, o que acaba gerarando uma boa carga de estudos aprendendo a melhor forma de escrever um código Kotlin.

Relação e Características Kotlin/Java

  • Interoperabilidade: Kotlin é totalmente interoperável com o Java, o que significa que o código de ambos pode ser utilizado no desenvolvimento do mesmo projeto. O mesmo vale para as bibliotecas importadas para projetos Java, que podem ser utilizadas em um projeto de Kotlin, dado que as APIs de ambas as linguagens são as mesmas.
  • Sintaxe: Ao mencionarmos a sintaxe, estamos nos referindo à característica de legibilidade do código que, se comparada a do Java, é mais simples e concisa, portanto, também mais vantajosa.
  • Null Safety: Essa característica do Kotlin, é um recurso da linguagem que é capaz de diferenciar referências que podem ser nulas daquelas que não podem ser, de acordo com o tipo definido no código.

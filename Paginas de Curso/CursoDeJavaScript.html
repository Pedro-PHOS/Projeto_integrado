JavaScript

JavaScript é uma linguagem leve, baseada em objetos que trabalha com funções de primeira classe. Ela é reconhecida por ser utilizada na criação de scripts para páginas Web, embora também seja usada também em vários outros ambientes sem browser. O JavaScript é uma linguagem baseada em protótipos, multi-paradigma e dinâmica, suportando estilos de orientação a objetos, imperativos e declarativos, como por exemplo a programação funcional.

A História por Trás do Nome...

O Javascript foi concebido em 1995 por Brendan Eich, com o nome de Mocha, que logo depois foi alterado para LiveScript, mas foi No final de 1995, por conta da ascensão do Java no mundo da programação que o fundador da linguagen resolveu mudar o nome para algo que fosse semelhante, a fim de que a atenção do público se voltasse mais para a nova tecnologia, assim nasceu o nome, Javascript.

O Fenômeno por Trâs da Ferramenta

O Javascript tornou possível a dinamização das antigas páginas de sites da internet. Foi através dessa linguagem, que foi possível criar interações, animações entre várias outras coisas que possibilitaram a assenção dos designs gráficos que existem hoje na criação de sites com HTML e CSS. Sua popularidade é decorrente do fato de ela ser uma linguagem extremamente flexível e multifacetada, fazendo com que milhares de pessoas desenvolvedoras se interessassem por ela, inclusive para escrever os primeiro códigos. Para termos uma ideia da versatilidade dessa linguagem, Javascript pode ser usado até mesmo no back-end de aplicações, atuando no processo de processamento de informações de um banco de dados, por exemplo.

Principais Vantagens e Desvantagens

Assim como é de praxe, nada no mundo é perfeito e por isso precisamos avaliar com calma e cuidado as principais vantagens e desvantagens das tecnologias que utilizamos no processo de desenvolvimento de software, para que possamos mensurar se o que está sendo empregado nesse processo, é de fato aquilo que necessitamos, por isso, separamos aqui as principais vantagens e desvantagens do JavaScript.


Vantagens

  • Flexibilidade e versatilidade: JavaScript é uma linguagem interpretada que segue o padrão de desenvolvimento web, o que significa que ela pode ser executada no contexto de diferentes navegadores, simplificando o processo de desenvolvimento. Outro ponto importante é que a linguagem pode ser executada no lado do servidor, o que permite que ela seja utilizada para criar diversas outras aplicações, tais jogos, aplicativos desktop e aplicativos mobile.

  • Alto nível: O JS é uma linguagem de alto nível que apresenta comandos em um idioma comum (inglês), o que facilita o desenvolvimento se comparada a linguagens de baixo nível que se utilizam de comandos com maior complexidade e termos desconhecidos/abreviações.

  • Linguagem Leve: Todo Navegador possui um interpretador apto a trabalhar com essa linguagem, o que faz com que todos consigam interpretar códigos desenvolvidos com JavaScript de forna rápida. Outro ponto que contribui com o fator de agilidade, é o fato de que o código é executado sem que haja a necessidade de utilizar um compilador ou interpretador externo, permitindo que alterações no código ou identificação de erros sejam feitas em tempo de execução.

Desvantagens

  • Segurança limitada: Um dos problemas das linguagens interpretadas é o fato do seu código ser aberto e disponível para inspeção no sistema que o interpreta. O JavaScript permite entender a lógica por trás de qualquer aplicação e, inclusive, copiar algumas partes dela. Por conta disso, a segurança acaba sendo um fator considerado como um problema para a linguagem, principalmente na construção de aplicações que têm acesso a fatores sensíveis, como a web cams de usuários, por exemplo.

  • Frameworks obsoletos: Embora a linguagem te oferça inúmeras ferramentas para solucão de problemas, infelizmente a realidade mostra que surgem novos frameworks com uma frequência muito alta, o que acaba por tornar os outros obsoletos frequentemente.

  • Diferentes interpretadores: A capacidade do JavaScript de ser interpretado por diversos navegadores é considerada como uma vantagem, porém, isso demanda que do desenvolvedor, muta atenção ao implementar novas funcionalidades. Isso por que, cada navegador utiliza o seu próprio motor para interpretar Javascript, o que em alguns casos pode ocasionar em interpretações diferentes do mesmo código.
